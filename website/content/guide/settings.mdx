<DocPage title="Settings" description="App preferences and configuration.">

Settings are opened from the **Command bar** (the bar with **Add**, **Plan**, **Publish**, the YouTube chip, and the Settings icon). Click the **Settings (gear)** icon on the right side of the Command bar to open a **dropdown menu** with: **Interface**, **Account**, **Custom AI**, **Diagnostics**, and **Developer mode**. Each item opens a separate dialog.

<StepBlock title="Step 1 — Open Settings menu" imageSrc="/docs/images/settings/01-settings-button.jpg" imageAlt="Settings (gear) in Command bar" caption="Screenshot 01 — Settings (gear) in Command bar: opens menu with Interface, Account, Custom AI, Diagnostics, Developer mode">
Click the **Settings** (gear) icon in the **Command bar**. The dropdown opens with **Interface**, **Account**, **Custom AI**, **Diagnostics**, and **Developer mode**. Choose one to open its dialog.
</StepBlock>

<StepBlock title="Step 2 — Interface" imageSrc="/docs/images/settings/02-interface.jpg" imageAlt="Interface dialog" caption="Screenshot 02 — Interface: UI size, theme, language, command bar position, panels layout">
Choose **Interface** to open the Interface dialog. You can set:
- **UI size** — Slider from 100% to 200% (with A− / A+ buttons).
- **Day / Night** — Theme switch (light/dark).
- **Language** — Dropdown for app language.
- **Command bar position** — **Top** or **Bottom** (where the main action bar appears).
- **Panels layout** — **Default** or **Swapped** (order of table and Details panel).
Click **Close** to apply. All settings are saved automatically. The **Details panel** width is not in this dialog — resize it by dragging the splitter between the table and the Details panel.
</StepBlock>

<StepBlock title="Step 3 — Account" imageSrc="/docs/images/settings/03-account.jpg" imageAlt="Account dialog" caption="Screenshot 03 — Account: Manage billing, Subscription (coming soon)">
Choose **Account** to open the Account dialog. It shows **Manage billing** and **Subscription** (both show “coming soon” when clicked). YouTube sign-in is **not** in this dialog: use the **YouTube chip** in the Command bar (click it) for **Test connection** and **Reconnect**. **Advanced:** If you manage your own OAuth credentials, follow **YouTube Connect & Upload** and add them via **Settings → Developer mode → Open User Data**.
</StepBlock>

<StepBlock title="Step 4 — Custom AI" imageSrc="/docs/images/settings/04-custom-ai.jpg" imageAlt="Custom AI dialog" caption="Screenshot 04 — Custom AI: presets and instructions">
Choose **Custom AI** to manage AI presets, custom instructions per platform, and description templates used when generating metadata. See **Presets** for details.
</StepBlock>

<StepBlock title="Step 5 — Diagnostics" imageSrc="/docs/images/settings/05-diagnostics.jpg" imageAlt="Diagnostics dialog" caption="Screenshot 05 — Diagnostics: connections, Auto Upload, jobs, system info">
Choose **Diagnostics** to open the Diagnostics dialog. It shows read-only status: **Connections** (YouTube: Connected / Not connected), **Auto Upload** (Enabled, and **Silent** when enabled), **Jobs** (Total count, Processing count), and **System info** (Platform, Architecture, Version). Use it to verify connection and environment.
</StepBlock>

<StepBlock title="Step 6 — Developer mode" imageSrc="/docs/images/settings/06-developer-mode.jpg" imageAlt="Developer mode dialog" caption="Screenshot 06 — Developer mode: Debug mode, Open UserData, Open outputs">
Choose **Developer mode** to open the Developer mode dialog. It shows a warning and: **Debug mode** (extra diagnostics) and **Developer Actions** like **Open User Data** (app data folder) and **Open outputs** (outputs folder). There is no separate “output path” setting in the app — you open the folder from here or from the **Details panel** (**Open outputs** when a row is selected).
</StepBlock>

**Persistence**
- **Interface** choices (UI size, theme, language, command bar position, panels layout, details panel width) are saved automatically and persist across restarts.
- **Developer** Debug mode is saved automatically.

**Common issues**
- **YouTube not in Account** — Connect or reconnect YouTube from the **YouTube chip** in the Command bar (Test connection, Reconnect).
- **Output folder** — You can change the Outputs folder in **Settings → Developer mode** under **Outputs folder** (Browse… / Reset). Open it via **Developer mode → Open outputs** or **Details panel → Open outputs**.
