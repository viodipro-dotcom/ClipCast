<DocPage title="Troubleshooting" description="Fix common issues and errors.">

How to fix common issues based on the app’s actual behavior.

<StepBlock title="Step 1 — Generate Metadata (pipeline) fails" imageSrc="/docs/images/troubleshooting/01-pipeline-log.jpg" imageAlt="Pipeline Log" caption="Screenshot 01 — Details panel: Activity and Show technical log">
If **Generate Metadata** fails, the app shows: *"Metadata generation failed (code: X). Check Technical Log for details."*  
**What to do:** Select the row, open the **Details** panel, and click **Show technical log** in the **Activity** section to see the error for that file.  
If it still fails, retry once and make sure the video file is accessible and you have free disk space. If needed, share the technical log with support.
</StepBlock>

<StepBlock title="Step 2 — YouTube connection fails" imageSrc="/docs/images/troubleshooting/02-youtube-connect.jpg" imageAlt="Connect YouTube" caption="Screenshot 02 — Connect YouTube and OAuth setup">
If **Connect YouTube** fails:  
- **"OAuth credentials missing"** — Follow **YouTube Connect &amp; Upload** to set up credentials. **Advanced:** If you manage your own OAuth credentials, add them via **Settings → Developer mode → Open User Data**.  
- **Error 401 / invalid_client** — Use an OAuth client type **Desktop app** (not Web application), copy Client ID and Secret fully, enable **YouTube Data API v3**, and configure the OAuth consent screen. See **YouTube Connect &amp; Upload** for the setup checklist.  
Use **Test connection** and **Reconnect** from the **YouTube chip** in the command bar (click the YouTube status) to verify credentials.
</StepBlock>

<StepBlock title="Step 3 — File missing on disk">
If a row shows **"File missing on disk. Re-import or remove this row."** (status **Error** in the Details **Activity** / technical log), the file was moved, renamed, or deleted. **Remove** that row from the table, then add the video again from its **new location**: use **Add (Files or Folder)** in the toolbar to open the **Add Videos** dialog and choose **Add Files** or **Add Folder**.
</StepBlock>

<StepBlock title="Step 4 — Upload failed">
If upload fails, check the **Activity** section in the **Details** panel and **Show technical log** for the selected row. Use **Retry upload** (for selected failed items) to try again. For YouTube quota issues, wait and retry later. Ensure **YouTube** is connected (Connect YouTube in the toolbar or Settings) and credentials are valid.
</StepBlock>

<StepBlock title="Step 5 — GPU detected, but ClipCast still uses CPU">
ClipCast may detect your NVIDIA GPU, but still fall back to CPU if no usable CUDA device is exposed by the current driver/runtime. GPU mode requires an NVIDIA GPU and a **recent NVIDIA driver** compatible with our CUDA 12.x stack.  
**Check:** Open **Developer Mode**, review **Selected device**, **CUDA smoke test**, and **Fallback reason**. Run <code>nvidia-smi</code>.  
If the driver is too old, **update the NVIDIA driver**, restart the PC, and test again. If GPU mode is unavailable, ClipCast falls back to CPU automatically.
</StepBlock>

<StepBlock title="Step 6 — App won't start" imageSrc="/docs/images/troubleshooting/05-app-wont-start.jpg" imageAlt="Main window" caption="Screenshot 05 — Visual C++ (Windows) or Security (macOS)">
On **Windows**, install the latest **Visual C++ redistributable**. On **macOS**, if the app is blocked, go to **System Preferences → Security &amp; Privacy** and allow the app.
</StepBlock>

**Common tools in the app**
- **Show technical log** — In the **Details** panel when a row is selected: **Activity** section → **Show technical log**. Shows the log for that row (including pipeline and upload messages).
- **Developer mode** — In the **Settings** (gear) menu: advanced tools like **Open User Data** and **Open outputs**.
- **Diagnostics** — In the **Settings** (gear) menu: shows **Connections**, **Auto Upload**, **Jobs**, and **System info** for support.
