<DocPage title="Add Files & Jobs" description="Import video files or folders and add rows to the Jobs table.">

You can add videos in two ways: via the **Add Videos** dialog (files or folder) or by **drag-and-drop**. Each added file becomes one **row** in the **Jobs table**. Duplicate paths (already in the table or in scheduled jobs) are skipped.

<StepBlock title="Step 1 — Open Add Videos" imageSrc="/docs/images/add-files-jobs/01-add-files-button.jpg" imageAlt="Add (Files or Folder) button in the top toolbar" caption="Screenshot 01 — Top toolbar: Add (Files or Folder) button">
In the **top toolbar**, click **Add (Files or Folder)** to open the **Add Videos** dialog.
</StepBlock>

<StepBlock title="Step 2 — Choose files or a folder" imageSrc="/docs/images/add-files-jobs/02-choose-mp4-files.jpg" imageAlt="Add Videos dialog with Files and Folder buttons" caption="Screenshot 02 — Add Videos dialog: Files or Folder">
In the **Add Videos** dialog you have two options (Windows cannot combine file and folder selection in one picker):

- **Files** — Opens the native **file picker** with multi-select. The picker is filtered to video types: **MP4, MOV, MKV, WEBM**. Select one or more files and confirm.
- **Folder** — Opens the **folder picker**. After you choose a folder, the app **scans** it for video files (MP4, MOV, MKV, WEBM, AVI) and adds them all.

The dialog closes after you confirm; new rows appear in the Jobs table.
</StepBlock>

<StepBlock title="Step 3 — Rows appear in the table" imageSrc="/docs/images/add-files-jobs/03-jobs-grid.jpg" imageAlt="Jobs table with video rows" caption="Screenshot 03 — Jobs table: each file appears as a row">
Each new file becomes one **row** in the **Jobs table** with default **Title** (filename), **Status** (e.g. Ready), and platform targets. If **Auto plan** is on, new rows get default targets and may get a **Schedule** (publish time) assigned automatically. If the app finds existing metadata in <code>yt_pipeline/outputs</code> for that file, the row status is updated to **Done** after a short check.
</StepBlock>

<StepBlock title="Step 4 — Select a row for details" imageSrc="/docs/images/add-files-jobs/ " imageAlt="Selected row and Details panel" caption="Screenshot 04 — Details panel shows selected row">
Click a **row** to select it. The **Details panel** on the right shows that row’s file path, metadata, schedule, and targets.
</StepBlock>

<StepBlock title="Step 5 — Edit title and description" imageSrc="/docs/images/add-files-jobs/05-edit-title-description.jpg" imageAlt="Details panel: title and description fields" caption="Screenshot 05 — Details panel: title and description">
In the **Details panel**, edit **Title** and **Description** as needed. After running the pipeline, you can refine the AI-generated metadata here.
</StepBlock>

<StepBlock title="Step 6 — Run the pipeline" imageSrc="/docs/images/add-files-jobs/06-run-pipeline.jpg" imageAlt="Generate button in Details panel" caption="Screenshot 06 — Details panel: Generate">
Click **Generate** in the **Details panel** (Metadata section) for the selected row, or select multiple rows, click **Publish**, choose **Metadata only**, then **Generate**. Outputs are written to <code>yt_pipeline/outputs</code> and the row status updates.
</StepBlock>

**Drag-and-drop** — You can also drop video files onto the main app area. Supported extensions for drop: **MP4, MOV, AVI, MKV, WEBM, FLV, WMV, M4V**. The same logic applies: one row per file, duplicates skipped.

**Sort order** — When you add multiple files at once, they are ordered according to the current **Sort by** setting (File column menu): **Added order**, **Name (A–Z)**, or **Creation date**. After adding, the table filter is set to **All** so new rows are visible.

**Common issues**
- **Unsupported formats** — The file picker only lists MP4, MOV, MKV, WEBM. Folder scan also includes AVI. Other formats are not added via the dialog; drag-and-drop accepts more (e.g. AVI, FLV, WMV, M4V).
- **Duplicate file** — If a file path is already in the table or in scheduled jobs, it is skipped (no second row).
- **Missing file** — If a file was moved or deleted, remove the row or re-add the file from its new location.
- **Many files** — You can add many at once; run the pipeline per row or in batch. See **Pipeline** for logs and output location.
